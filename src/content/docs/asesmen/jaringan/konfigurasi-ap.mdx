## Konfigurasi Router 2-10 (Access Point Mode)

### Konsep Access Point Mode

Dalam mode Access Point:

- Router berfungsi sebagai WiFi Access Point saja
- Tidak melakukan routing atau NAT
- DHCP Server tetap aktif (masing-masing AP punya DHCP pool sendiri)
- Semua AP terhubung ke Router 1 via kabel LAN

### Step 1: Akses Router (Default Settings)

1. Hubungkan Router ke laptop via kabel LAN
2. Akses `http://192.168.0.1`
3. Login: admin/admin

### Step 2: Ubah IP Address

**Untuk Router 2:**

1. Masuk ke **Network â†’ LAN**
2. Ubah:
   ```
   IP Address: 192.168.1.2
   Subnet Mask: 255.255.255.0
   ```
3. **Save/Apply**
4. **Setelah restart, akses:** `http://192.168.1.2`

3. Atau abaikan WAN settings (tidak dipakai)

### Step 4: Konfigurasi DHCP Server

### Step 10: Physical Connection

## Testing dan Verifikasi

### Test Checklist

#### Test Router 1 (Main Router)

- [ ] Dapat diakses di http://192.168.1.1
- [ ] DHCP memberikan IP 192.168.1.101-120
- [ ] WiFi ASESMEN-CBT dapat diakses
- [ ] Password WiFi bekerja dengan benar
- [ ] Ping ke 192.168.1.1 berhasil
- [ ] Ping ke 192.168.1.100 (server) berhasil

#### Test Router 2-10 (Access Point)

- [ ] Masing-masing dapat diakses di IP-nya (192.168.1.2-10)
- [ ] DHCP memberikan IP sesuai pool yang dikonfigurasi
- [ ] WiFi dapat diakses dengan SSID yang benar
- [ ] Password WiFi sama dengan Router 1 (jika pakai SSID sama)
- [ ] Device yang connect dapat ping ke 192.168.1.1
- [ ] Device yang connect dapat ping ke 192.168.1.100 (server)
- [ ] Device dapat akses aplikasi CBT

### Testing Procedure

1. **Test Satu Router per Waktu**

   - Connect laptop/HP ke WiFi router yang akan ditest
   - Cek IP Address yang didapat (harus sesuai DHCP pool)
   - Ping ke gateway (192.168.1.1)
   - Ping ke server (192.168.1.100)
   - Buka browser, akses IP server: http://192.168.1.100

2. **Test Load dengan Multiple Devices**

   - Connect 5-10 device ke satu router
   - Pastikan semua dapat IP dan akses server
   - Cek performa (speed dan latency)

3. **Test Roaming (Jika Pakai SSID Sama)**
   - Connect ke Router 2
   - Berjalan ke area Router 3
   - Cek apakah otomatis pindah (roaming) tanpa putus koneksi

## Troubleshooting

### Problem: Tidak Bisa Akses Router Setelah Ubah IP

**Solusi:**

1. Reset IP laptop ke DHCP auto
2. Atau set IP manual:
   ```
   IP: 192.168.1.200
   Subnet: 255.255.255.0
   Gateway: 192.168.1.1
   ```
3. Akses router di IP baru-nya

### Problem: Device Tidak Dapat IP dari DHCP

**Solusi:**

1. Cek DHCP Server di router adalah "Enabled"
2. Cek DHCP pool tidak bentrok dengan IP lain
3. Cek lease time tidak terlalu pendek
4. Restart router
5. Release/Renew IP di device (disable/enable WiFi)

### Problem: Bisa Connect WiFi tapi Tidak Bisa Akses Server

**Solusi:**

1. Ping ke gateway (192.168.1.1) - jika gagal, masalah di router
2. Ping ke server (192.168.1.100) - jika gagal, cek:
   - Kabel LAN dari AP ke Router 1
   - Server sudah hidup dan web service running
   - Firewall di server tidak block akses
3. Cek DNS setting di DHCP (harus 192.168.1.100)

### Problem: WiFi Lambat atau Sering Putus

**Solusi:**

1. Cek interference dengan WiFi Analyzer
2. Ubah channel WiFi (gunakan channel 1, 6, atau 11)
3. Reduce transmit power jika terlalu banyak overlap
4. Cek jumlah connected device tidak melebihi 25
5. Restart Access Point

### Problem: Lupa Password Admin Router

**Solusi:**

1. Reset router ke factory default:
   - Tekan dan tahan tombol **Reset** selama 10 detik
   - Router akan restart dengan default setting
2. Konfigurasikan ulang dari awal
3. Atau restore dari backup file .bin
