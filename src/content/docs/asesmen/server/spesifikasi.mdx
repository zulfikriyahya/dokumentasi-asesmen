# Spesifikasi Server Asesmen CBT

## Pendahuluan

Dokumen ini menjelaskan spesifikasi minimum dan rekomendasi untuk server yang akan digunakan dalam sistem Computer Based Test (CBT) offline untuk melayani hingga 200 perangkat siswa secara bersamaan.

## Spesifikasi Minimum Hardware

### Processor

- **Minimum:** Intel Core i5 generasi ke-8 atau AMD Ryzen 5 2600
- **Rekomendasi:** Intel Core i7 generasi ke-10 atau AMD Ryzen 7 3700X
- **Core:** Minimal 4 core / 8 thread
- **Alasan:** Memproses request simultan dari 200+ perangkat memerlukan multi-threading yang baik

### RAM (Memory)

- **Minimum:** 16 GB DDR4
- **Rekomendasi:** 32 GB DDR4 atau lebih
- **Alasan:** Database MySQL dan Web Server Apache/Nginx membutuhkan memory yang cukup untuk caching dan menangani koneksi simultan

### Storage

- **Minimum:**
  - 256 GB SSD untuk sistem operasi dan aplikasi
  - 500 GB HDD untuk database dan backup
- **Rekomendasi:**
  - 512 GB NVMe SSD untuk sistem operasi dan aplikasi
  - 1 TB SSD/HDD untuk database dan backup
- **Alasan:** SSD meningkatkan kecepatan read/write database saat ratusan siswa mengakses soal secara bersamaan

### Network Interface Card (NIC)

- **Minimum:** 1 Gigabit Ethernet (1000 Mbps)
- **Rekomendasi:** 1 Gigabit Ethernet atau lebih tinggi
- **Alasan:** Bandwidth tinggi untuk melayani transfer data ke 200+ perangkat

### Power Supply

- **Minimum:** 500W 80+ Bronze
- **Rekomendasi:** 650W 80+ Gold dengan UPS backup 1500VA
- **Alasan:** Stabilitas daya sangat penting untuk menghindari shutdown mendadak saat asesmen berlangsung

## Spesifikasi Software

### Sistem Operasi

**Debian 13 (Trixie) - Sistem yang Digunakan**

- **Versi:** Debian 13 "Trixie"
- **Lisensi:** Gratis (Open Source)
- **Keunggulan:**
  - Sangat stabil dan reliable untuk production server
  - Performa lightweight dan efisien
  - Security update reguler dan cepat
  - Package management dengan APT yang mature
  - Dokumentasi lengkap dan komunitas besar
  - Cocok untuk server yang harus berjalan 24/7
- **Arsitektur:** amd64 (64-bit) atau arm64 (untuk ARM-based server)

**Catatan:** Debian 13 adalah versi terbaru (testing/unstable saat dokumentasi ini dibuat). Untuk production yang lebih stabil, bisa menggunakan Debian 12 "Bookworm" (stable).

### Database Server

- **Software:** MariaDB Server (fork dari MySQL, lebih cocok untuk Debian)
- **Versi:** MariaDB 10.11+ (tersedia di Debian 13 repository)
- **Kebutuhan Storage Database:** 50-100 GB (tergantung jumlah bank soal)
- **Alternatif:** MySQL Server 8.0+ (jika diperlukan kompatibilitas khusus)

### Web Server

**Apache2 (Rekomendasi untuk Debian + PHP)**

- **Versi:** Apache 2.4+ (default di Debian 13)
- **Module:** mod_rewrite, mod_ssl, mod_php atau php-fpm
- **Keunggulan:**
  - Integrasi sempurna dengan PHP
  - .htaccess support untuk rewrite rules
  - Konfigurasi per-directory yang fleksibel

**Nginx (Alternatif)**

- **Versi:** Nginx 1.22+ (tersedia di Debian 13)
- **Keunggulan:** Performa lebih baik untuk static files dan concurrent connections
- **Catatan:** Memerlukan PHP-FPM (tidak bisa mod_php)

### Runtime Environment

- **PHP:** Versi 8.2 atau 8.3 (tersedia di Debian 13 repository)
- **Extensions yang diperlukan:**
  - php-mysql atau php-mysqli (untuk koneksi ke MariaDB/MySQL)
  - php-mbstring (untuk multi-byte string handling)
  - php-xml (untuk XML processing)
  - php-curl (untuk HTTP requests jika diperlukan)
  - php-gd (untuk image processing)
  - php-zip (untuk compression/extraction)
  - php-json (untuk JSON handling)
  - php-intl (untuk internationalization - opsional)
- **Catatan:** Versi PHP harus disesuaikan dengan requirement aplikasi CBT yang digunakan.

## Konfigurasi Jaringan Server

### IP Address Configuration

```
IP Address: 192.168.1.100
Subnet Mask: 255.255.255.0
Gateway: 192.168.1.1
DNS Server: 8.8.8.8, 8.8.4.4 (atau DNS lokal jika ada)
```

### Firewall Rules (UFW - Uncomplicated Firewall)

```bash
# Install UFW (jika belum ada)
sudo apt install ufw

# Default policies
sudo ufw default deny incoming
sudo ufw default allow outgoing

# Allow SSH (untuk remote management)
sudo ufw allow 22/tcp

# Allow HTTP dan HTTPS untuk aplikasi CBT
sudo ufw allow 80/tcp
sudo ufw allow 443/tcp

# Enable firewall
sudo ufw enable

# Check status
sudo ufw status verbose
```

**Port Configuration:**

- **Port 22 (SSH):** Open untuk remote management (opsional, bisa dibatasi ke IP admin tertentu)
- **Port 80 (HTTP):** Open untuk akses aplikasi CBT
- **Port 443 (HTTPS):** Open untuk akses aplikasi CBT dengan SSL (opsional)
- **Port 3306 (MariaDB/MySQL):** Automatically blocked from external (hanya listen di localhost)

**Catatan Keamanan:**

- MariaDB/MySQL default hanya listen di 127.0.0.1 (localhost)
- Tidak perlu open port 3306 ke network karena web server dan database di server yang sama
- Untuk keamanan ekstra, SSH bisa dibatasi hanya dari IP administrator tertentu

## Estimasi Kapasitas

### Concurrent Users

- **Maksimal:** 200-220 perangkat siswa
- **Rata-rata Load:** 180-200 active sessions
- **Buffer:** 10% untuk administrator dan pengawas

### Bandwidth Requirement

- **Per Device:** ~1-2 Mbps (untuk load soal dan submit jawaban)
- **Total Bandwidth:** Minimal 200-400 Mbps internal network
- **Catatan:** Ini adalah bandwidth internal (server ke client), bukan internet

### Database Size Estimation

- **Per Siswa/Session:** ~5-10 MB
- **Bank Soal (dengan gambar):** ~50-200 MB per paket soal
- **Log System:** ~20-50 MB per hari asesmen
- **Total Estimation:** 50-150 GB untuk satu tahun akademik

## Checklist Persiapan Hardware

- [ ] Server dengan spesifikasi minimum terpenuhi
- [ ] RAM 16 GB atau lebih terinstal
- [ ] SSD untuk sistem operasi (min 256 GB)
- [ ] HDD/SSD untuk database (min 500 GB)
- [ ] Network Card Gigabit berfungsi normal
- [ ] UPS dengan kapasitas minimal 1500VA
- [ ] Kabel LAN Cat6 untuk koneksi ke Router utama
- [ ] Cooling system (kipas/AC) memadai untuk ruang server
- [ ] Akses fisik yang aman (ruangan terkunci)

## Rekomendasi Tambahan

### Backup System

- **Backup Harian:** Backup database setiap hari setelah asesmen selesai
- **Backup Mingguan:** Full system backup setiap minggu
- **Media Backup:** External HDD minimal 1 TB atau NAS

### Monitoring

- **Temperature:** Monitor suhu CPU dan HDD
- **Disk Space:** Monitor space yang tersisa minimal 20%
- **Network:** Monitor koneksi dan latency
- **Service:** Monitor status MySQL dan Web Server

### Security

- **Firewall:** UFW (Uncomplicated Firewall) - default di Debian
- **Antivirus:** ClamAV (opsional untuk scanning uploaded files)
- **Fail2ban:** Untuk proteksi terhadap brute-force attacks
- **User Access:**
  - User root: disabled untuk SSH login (gunakan sudo)
  - Buat user admin terpisah dengan sudo privileges
  - User www-data: untuk web server (default)
- **Password Policy:**
  - Password minimal 12 karakter
  - Kombinasi huruf besar, huruf kecil, angka, dan simbol
  - Gunakan SSH key authentication untuk remote access (lebih aman dari password)
- **System Updates:**
  ```bash
  # Update system secara berkala
  sudo apt update && sudo apt upgrade -y
  sudo apt autoremove -y
  ```

## Troubleshooting Common Issues

### Server Lambat

```bash
# Cek penggunaan RAM
free -h
htop  # atau top jika htop belum terinstall

# Cek penggunaan CPU
htop
mpstat  # dari package sysstat

# Cek penggunaan Disk I/O
iostat -x 1  # dari package sysstat
iotop  # monitoring I/O per process

# Cek koneksi jaringan
ip addr show
ping 192.168.1.1
ifconfig  # atau gunakan ip command

# Restart services jika perlu
sudo systemctl restart mariadb
sudo systemctl restart apache2
# atau
sudo systemctl restart nginx php8.2-fpm
```

### Koneksi Database Error

```bash
# Cek status MariaDB service
sudo systemctl status mariadb

# Restart MariaDB jika perlu
sudo systemctl restart mariadb

# Cek log error MariaDB
sudo tail -f /var/log/mysql/error.log

# Cek kredensial database di konfigurasi aplikasi
# File konfigurasi biasanya di:
# - /var/www/html/config/database.php
# - /var/www/html/.env
# - atau sesuai dengan aplikasi CBT yang digunakan

# Test koneksi database manual
mysql -u username -p -h localhost database_name

# Cek max_connections di MariaDB
mysql -u root -p -e "SHOW VARIABLES LIKE 'max_connections';"
```

### Disk Space Full

```bash
# Cek disk space
df -h

# Cek directory yang besar
du -sh /var/log/*
du -sh /var/www/*
du -sh /tmp/*

# Hapus log lama
sudo find /var/log -type f -name "*.log" -mtime +30 -delete
sudo journalctl --vacuum-time=7d  # Hapus systemd journal > 7 hari

# Hapus backup lama (manual, sesuaikan path)
# Contoh: rm -rf /backup/old_backups/*

# Hapus cache APT
sudo apt clean
sudo apt autoclean

# Hapus package yang tidak diperlukan
sudo apt autoremove -y

# Hapus temporary files
sudo rm -rf /tmp/*
```
